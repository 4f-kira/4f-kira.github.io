攻击思路

  1. 本地生成序列化session文件
  2. 跟进session文件名规则计算出文件名
  3. 利用mysql的注入，将文件写入/tmp/ffff目录
  4. 访问index的时，修改自己的session为之前我们定义的值
  5. 触发open_session中的self.cache.get，进行反序列化攻击
